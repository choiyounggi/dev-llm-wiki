# dev-llm-wiki

[English](README.md) | **한국어**

**LLM 에이전트가 작업 컨텍스트로 로드하도록 설계된**, 케이스 라우팅형 개발
베스트프랙티스·엣지케이스 위키 — 당신의 태스크를 수행하는 에이전트가 정확히
해당되는 지침만 끌어오고, 그 외에는 아무것도 로드하지 않게 합니다.

[Andrej Karpathy의 LLM-wiki 패턴](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
위에 만들어졌습니다: 에이전트가 유지하는 마크다운 페이지, 에이전트를 규율하는
스키마, 그리고 지식이 썩는 대신 복리로 쌓이게 하는 ingest/query/lint 워크플로우.

## 왜 존재하는가

에이전트는 *특정 상황*에 맞는 프랙티스가 컨텍스트에 있을 때 실수를 덜 합니다 —
5,000단어짜리 스타일 가이드도, "톱 10 팁" 글도 아니고요. 그러려면 대부분의
지식 베이스에 없는 세 가지 속성이 필요합니다:

1. **케이스 라우팅.** 모든 페이지가 *언제 적용되는지*를 선언합니다. 인덱스는
   트리거의 지도라서, 에이전트(또는 사람)가 `INDEX.md` → 도메인 → 페이지로
   내려가며 매칭되는 것만 로드합니다. 벌크 컨텍스트도, 노이즈도 없습니다.
2. **긍정 유도형 지침.** 모든 지시는 "상황 X에서는 Y를 하라"입니다. 금지는
   반드시 대체 행동과 짝지어질 때만 허용됩니다 — 대체 없는 "하지 마라"는 다음
   단계를 독자의 상상에 맡기고, LLM에게 그 지점이 바로 할루시네이션의
   시작점입니다.
3. **근거 규율.** 모든 페이지가 출처를 인용하고 신뢰도 태그(`verified` /
   `field-tested` / `unverified`)를 답니다. lint가 무출처 주장을 사냥하므로,
   그럴듯하게 지어낸 것이 조용히 "지식"이 되지 못합니다.

## 에이전트와 함께 쓰기

```bash
git clone https://github.com/choiyounggi/dev-llm-wiki.git
```

- **Claude Code / Codex / Cursor 등**: 레포에 `AGENTS.md`(스키마)와 `CLAUDE.md`가
  들어 있어, 레포를 열면 에이전트가 라우팅 규약을 배웁니다. 에이전트를 레포에
  향하게 하거나, 워크스페이스에 심링크해 참조하세요.
- **수동**: [INDEX.md](INDEX.md)에서 시작해 "route here when" 라인을 따라가세요.

세 가지 운영 + 두 가지 개발 워크플로우 (`skills/`):

| 워크플로우 | 용도 |
|-----------|-----|
| `skills/ingest` | 지식 추가 — 배운 교훈, 엣지케이스, 출처 있는 프랙티스. 기존 페이지에 병합하고, 출처와 긍정 프레이밍을 강제 |
| `skills/query` | 위키에서 인용과 함께 질문에 답변; 반복되는 종합은 페이지로 되돌려 등록 |
| `skills/lint` | 건강 점검 — 무출처 주장, 대체 없는 금지, 깨진 링크, 낡은 페이지 |
| `skills/plan` | 유능한 모델용: 모든 설계 결정을 (위키에 근거해) 내리고, 작은 모델이 태스크 파일 + 매핑된 위키 페이지만으로 실행할 수 있는 크기의 순서 있는 태스크로 분해 |
| `skills/implement` | 계획된 태스크 하나를 실행하는 작은 모델용: 태스크가 명시한 것만 로드, 즉흥 금지(빠진 결정은 추측이 아니라 BLOCKED 보고), 검증, 보고 |

## 현황

시드된 도메인: **databases** (인덱싱, 쿼리 최적화, 스키마 설계, 트랜잭션 —
15페이지). 나머지 도메인(backend, frontend, infrastructure, testing, debugging,
security, platforms, mobile)은 골격 상태이며 ingest로 성장합니다.

기여 환영: 에이전트로 ingest 워크플로우를 돌리거나, `templates/page.md`와
`AGENTS.md`의 규칙을 따르는 PR을 열어주세요.
